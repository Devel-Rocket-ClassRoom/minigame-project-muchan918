# 7Days — 개발 일정표

> 개발 기간: 3주 (2026.05.17 ~ 2026.06.06)
> 플랫폼: Android (Unity 6.3, URP)
> 목표: Play Store 출시
> **최신 갱신: 2026-05-21 — 1주차 핵심 루프 목표 변경 반영**

---

## 📌 1주차 핵심 목표 (변경됨)

기존 일정에서는 제작/동물 AI/조리까지 1주차에 포함되어 있었으나,
**1주차에는 게임의 핵심 루프 1회분만 동작하도록 범위를 축소**하였다.

> **새 1주차 핵심 루프**
> **채집 → 아이템 습득 → 7일차 이벤트 발생 → 생존 여부 결정**

핵심 루프가 처음부터 끝까지 한 사이클 동작하는 것을 1주차 완료 기준으로 둔다.
동물 AI / 조리 / 제작 / 장비 등은 2주차 이후로 이연한다.

### Day 2 (05.18) ✅

- [x] 낮/밤 타이머 시스템 (날짜 카운트 포함)
- [ ] 낮↔밤 전환 연출 (#6 OPEN — 1주차 완료 전 마무리)
- [x] 취침 시스템 (오두막 근처 밤 → 페이드 → 다음 날)

### Day 3 (05.19) ✅
- [x] 자원 데이터 구조 설계 (ResourceData ScriptableObject)
- [x] 채집 시스템 (나무 / 돌 / 풀 / 열매)

### Day 4 (05.20) ✅
- [x] 아이템 데이터 구조 (ItemData ScriptableObject)
- [x] 인벤토리 시스템 (슬롯 / 추가·제거)
- [x] 체력(HP) 시스템 기초 + UI 체력바

### Day 5 (05.21) — **오늘 / 진행 중**
> 핵심 루프 후반부의 시작점. **7일차 이벤트가 동작할 수 있는 토대를 구축**한다.

- [ ] **7일차 이벤트 트리거** — DayManager가 Day==7 도달 시 이벤트 발화
- [ ] **제단(Altar) 오브젝트 배치** — 마을 입구에 인터랙션 가능한 제단 프리팹 설치
- [ ] **공물 데이터 구조 설계** — 1주차에는 신 1명 + 고정 공물 1세트 (예: 나무 5, 돌 3, 풀 2)
  - 추후 확장을 위해 ScriptableObject 기반으로 정의
- [ ] (선택) 배고픔(Hunger) 게이지 기초 — 생존 판정 보조 용도 (간소화 가능)

### Day 6 (05.22) — 제사/공물 제출 + 판정
- [ ] **제단 상호작용** → 공물 제출 UI 열기
- [ ] **공물 제출 창** — 요구 목록 / 보유량 / 누적 제출량 표시, 부분 제출 가능
- [ ] **공물 충족 여부 판정 로직** — 모든 요구 채워지면 충족 플래그 ON
- [ ] 제출 후 인벤토리에서 해당 재료 차감

### Day 7 (05.23) — 7일차 결과 & 1주차 통합 테스트
- [ ] **7일차 도달 시 결과 분기 처리**
  - 충족 → "부족에게 인정받았다" 텍스트 + 다음 사이클 예고 (또는 게임 클리어)
  - 미달 → 제물 연출(간단) + 게임 오버 텍스트
- [ ] **결과 화면 UI** — 최소한의 텍스트 + 재시작/메인 메뉴 버튼
- [ ] **1주차 통합 테스트** — 채집 → 인벤토리 → 제단 제출 → 7일차 판정 1회 완주
- [ ] 발견된 크리티컬 버그 즉시 수정

> 💡 1주차 완료 기준: **위 핵심 루프가 버그 없이 1회 이상 완주 가능**한 상태.
