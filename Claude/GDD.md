# 7Days — Game Design Document

> 레퍼런스: Don't Starve  
> 장르: 생존 액션  
> 시점: 3D 탑뷰  
> 개발 기간: 3주 (Unity 6.3)

---

## 1. 게임 개요

**7Days**는 야생에서 자원을 수집하고 기지를 성장시키며, 주기적으로 발생하는 재난과 이벤트를 대비해 생존하는 **생존 액션 게임**이다.

플레이어는 낮 동안 맵을 탐험하며 나무, 돌, 철광석 등의 자원을 채집하고, 동물 사냥 및 제작 시스템을 통해 기지를 강화한다. 밤이 되면 위험 요소가 증가하며, 제한된 시간 안에 기지로 복귀해야 한다.

게임은 **7일마다 발생하는 특수 이벤트**(폭풍, 혹한, 몬스터 습격 등)를 중심으로 진행되며, 플레이어는 자원 관리와 기지 운영을 통해 점점 더 강해지는 재난을 버텨야 한다. 이벤트를 생존하면 더 강한 이벤트가 해금되고, 포탈 제작을 통해 더 위험한 맵으로 진행할 수 있다.

---

## 2. 핵심 게임 루프

```
1일차 시작 (오두막에서 시작)
    │
    ▼
다가올 7일차 재난 이벤트 정보 전달
    │
    ▼
[낮] 탐험 & 채집 사이클
 ├── 나무 / 돌 / 광석 / 식량 수집
 ├── 기지 강화 / 장비 제작
 └── 텐트 설치 (임시 거점)
    │
    ▼
[밤] 기지 복귀 (위험도 증가)
    │
    ▼
7일째 — 특수 재난 이벤트 발생
    │
  생존?
 ├── YES → 더 강한 이벤트 해금 / 포탈 제작 → 다음 맵으로 이동
 └── NO  → 게임 오버
    │
    ▼
최종 재난 이벤트 → 생존 시 게임 승리
```

## 3. 구현 범위

### 3-1. 핵심 시스템

| 시스템 | 설명 |
|--------|------|
| 플레이어 이동 & 카메라 | 탑다운 이동, 카메라 추적 |
| 낮/밤 시간 시스템 | 시간 흐름, 낮↔밤 전환, 위험도 변화 |
| 채집 시스템 | 나무 / 돌 / 광석 채집 인터랙션 |
| 아이템 & 인벤토리 | 아이템 획득, 슬롯 관리 |
| 제작(Crafting) 시스템 | 자원 소모 → 도구/장비 제작 |
| 기지(집) 시스템 | 기지 건설, 강화, 안전 지역 판정 |
| 체력 & 생존 시스템 | HP, 배고픔, 식량 소모 |

### 3-2. 탐험 & 생존 요소

| 요소 | 설명 |
|------|------|
| 야생 동물 & 몬스터 AI | 상태 머신 기반 행동 (배회, 추적, 공격, 도망) |
| 위험 지역 탐험 | 날짜 경과에 따라 더 위험한 지역과 자원 등장 |
| 밤 시간 위험도 증가 | 야간 몬스터 활성화, 시야 제한 |
| 기지 방향 UI | 화면 가장자리에 기지 방향 표시 |
| 텐트 시스템 | 임시 거점 설치 (야간 안전 확보) |

### 3-3. 이벤트 시스템

| 이벤트 | 설명 |
|--------|------|
| 몬스터 습격 | 기지를 향해 몬스터 웨이브 발생 |
| 혹한 | 시간 경과 시 체온 감소, 난방 장치 필요 |
| 폭풍 | 이동 속도 감소, 시야 제한, 기지 내구도 피해 |
| 독안개 | 지속 HP 감소, 마스크 제작으로 방어 |
| 난이도 증가 | 이벤트 생존 시 더 강한 이벤트 해금 |
| 최종 이벤트 | 마지막 맵의 최종 재난 이벤트 |

### 3-4. 성장 요소

- 도구 업그레이드 (기본 → 강화 → 최상위 티어)
- 기지 강화 (내구도, 방어력, 저장 공간 확장)
- 자원 저장 및 관리
- 포탈 제작 → 더 강한 맵으로 이동

## 4. 포트폴리오 핵심 요소

### 4-1. 시간 기반 이벤트 시스템
낮/밤 변화와 날짜 시스템을 기반으로 특정 날짜마다 재난 이벤트가 발생하는 구조를 구현하여 게임 진행 흐름과 긴장감을 설계.

### 4-2. 데이터 기반 확장 구조
재난 이벤트, 몬스터, 자원 등을 ScriptableObject 기반으로 설계하여 새로운 콘텐츠를 쉽게 추가 가능한 구조로 구현.

### 4-3. 세이브 / 로드 시스템
플레이어의 진행 상황을 저장하고 로드하는 시스템 (저장 단위: 게임 내 하루).

### 4-4. 추상화 구조 설계
자원, 몬스터, 이벤트 등의 공통 동작을 추상화(인터페이스 / 베이스 클래스)하여 새로운 콘텐츠를 쉽게 추가할 수 있도록 구조를 설계. 유지보수성과 확장성을 고려한 게임 시스템 구현 경험을 보여주는 것을 목표.

### 4-5. AI 상태 머신
동물과 몬스터의 행동을 상태 머신(State Machine) 기반으로 구현하여 상황에 따라 자연스럽게 행동하도록 설계.

### 4-6. 치팅 툴 시스템
이벤트 상황을 원하는 대로 재현할 수 있는 인게임 치팅 툴 구현으로 QA 및 테스트 효율 향상.
