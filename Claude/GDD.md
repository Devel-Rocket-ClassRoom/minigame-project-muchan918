# 7Days — Game Design Document

> 레퍼런스: Don't Starve
> 장르: 생존 / 탐험 / 거점 강화
> 시점: 3D 탑뷰
> 플랫폼: Android (Unity 6.3, URP)
> 개발 기간: 3주 (2026.05.17 ~ 2026.06.06)
> **최신 갱신: 2026-05-21 — 탈출 컨셉 + 시설 업그레이드 시스템으로 전면 재구성**

---

## 1. 게임 개요

**7Days**는 신비로운 부족 마을 외곽에 갇혀, **7일마다 부족이 요구하는 공물을 바치며 살아남고**, 그 사이 **자기 거점(오두막 영역)을 강화해 탈출 수단인 "신비로운 돌"을 제작해 부족의 굴레에서 벗어나는** 생존·탐험 게임이다.

플레이어는 매 사이클(7일) 동안 야생에서 자원을 채집하고, 거점의 시설(작업대 / 가마솥 / 저장소)을 점진적으로 업그레이드하며, 마지막에는 **작업대 최종 단계에서만 제작 가능한 "신비로운 돌"을 제단에 봉헌해 탈출 엔딩**을 본다. 공물을 못 바치면 제물로 바쳐져 게임 오버.

매 사이클 종료(7일째 밤) 후 다음 날 아침 **야생 지역이 재생성**되어 자원이 리셋된다. 단 오두막 영역(시설·저장소·인벤토리)은 영구 유지된다.

---

## 2. 핵심 게임 루프

```
1일차 시작 (오두막에서 시작 — 시설 자리는 빈 표지판으로 표시됨)
    │
    ▼
[낮] 야생 탐험 & 채집
 ├── 일반 자원 (나무 / 돌 / 풀 / 점토 등)
 ├── 식량 자원 (열매 / 버섯)
 └── 희귀 자원 (먼 지역 — 시설 T3 / 신비로운 돌 재료)
    │
    ▼
[오두막 영역] 자원 분배 — "공물 vs 시설 업그레이드"
 ├── 공물 보관 (제단 제출용)
 ├── 시설 건설/업그레이드 (작업대 / 가마솥 / 저장소)
 └── 저장소에 누적 (다음 사이클 대비)
    │
    ▼
[밤] 오두막 취침 → 다음 날 (체력 회복)
    │
    ▼
7일째 — 제단에 공물 제출 → 7일차 평가
 ├── 공물 충족 → 다음 사이클 진행 → 야생 재생성
 └── 미달        → 제물로 바쳐짐 → 게임 오버
    │
    ▼
... 사이클 반복 + 시설 점차 업그레이드 ...
    │
    ▼
작업대 T3 도달 + 희귀 재료 모음 → 신비로운 돌 제작
    │
    ▼
제단에 신비로운 돌 봉헌 → 탈출 엔딩
```

## 3. 구현 범위

### 3-1. 핵심 시스템

| 시스템 | 설명 | 우선순위 |
|--------|------|----------|
| 플레이어 이동 & 카메라 | 탑다운 이동, 카메라 추적 | 🔴 1주 (완료) |
| 낮/밤 시간 시스템 | 시간 흐름, 낮↔밤 전환, 날짜 카운트 | 🔴 1주 (완료) |
| 채집 시스템 | 자원 오브젝트 인터랙션, 드롭 | 🔴 1주 (완료) |
| 아이템 & 인벤토리 | 슬롯 관리, 줍기/추가/제거 | 🔴 1주 (완료) |
| 체력(HP) 시스템 | HP 관리 + UI, 사망 처리 | 🔴 1주 (완료) |
| 제단 & 공물 제출 | 인터랙션 → 제출 UI → 충족 판정 | 🔴 1주 (작업 중) |
| 7일차 평가 (TributeEvent) | Day == 7 도달 시 충족/미달 판정 | 🔴 1주 (작업 중) |
| 7일차 결과 화면 | 생존(통과) / 제물(게임 오버) | 🔴 1주 (작업 중) |
| **시설 시스템 — 건설/업그레이드** | **작업대 / 가마솥 / 저장소 T1~T3** | 🟡 **2주** |
| **요리 시스템** | **가마솥에서 단계별 음식 제작** | 🟡 **2주** |
| 야생 영역 재생성 | 사이클 종료 시 자원 리스폰 | 🟡 2주 |
| 배고픔(Hunger) 시스템 | 시간 감소, 섭취 회복, 취침 패널티 | 🟡 2주 |
| **신비로운 돌 제작** | **작업대 T3에서 희귀 재료 조합** | 🟢 **3주** |
| **신비로운 돌 봉헌 & 탈출 엔딩** | **제단 UI 분기 + 엔딩 연출** | 🟢 **3주** |
| (선택) 동물 / 사냥 미니 버전 | 소형 동물 사냥 + 드롭 (전투는 액션 1버튼 휘두름) | 🟢 3주 |
| (선택) 치팅 툴 | QA용 자원/사이클 조작 | 🟢 3주 |

### 3-2. 공간 이원화 — 오두막 영역 vs 야생 영역

| 영역 | 영구성 | 설치물 | 자원 | 위험도 |
|------|--------|--------|------|--------|
| **오두막 영역** | 영구 | 시설(영구) + 저장소 누적 | 거의 없음 | 안전 |
| **야생 영역** | 사이클마다 재생성 | 텐트 등 임시 시설만 | 풍부 (멀수록 희귀) | 멀수록 위험 |

### 3-3. 탐험 & 생존 요소

| 요소 | 설명 |
|------|------|
| 위험 지역 탐험 | 먼 곳에 희귀 재료 (시설 T3 / 신비로운 돌용) |
| 사이클 압박 | 사이클이 진행될수록 공물 요구량 증가 (소프트 한계) |
| 자원 배분 갈등 | 같은 자원이 공물용/시설용으로 경쟁 |
| (선택) 동물 채집 | 소형 동물 = 식량/가죽 보조 자원 |

> 전투는 의도적으로 최소화 — 액션 버튼 1회 휘두름이 전부. 보스/적대 AI는 없음.

## 4. 포트폴리오 핵심 요소

### 4-1. 자원 경쟁 기반 핵심 루프
공물(생존)과 시설 업그레이드(탈출)가 같은 자원을 두고 경쟁하는 구조로, 매 사이클의 자원 배분이 의미 있는 결정이 되도록 설계.

### 4-2. 데이터 기반 확장 구조
시설 / 요리 / 자원 / 공물 요구를 ScriptableObject 기반으로 정의해 확장성과 유지보수성을 확보.

### 4-3. 공간 이원화 디자인
오두막 영역(영구) ↔ 야생 영역(사이클 재생성) 분리로 거점감과 탐험감을 모두 살림.

### 4-4. 단일 진엔딩 + 단순한 실패 엔딩
복잡한 분기 대신 명확한 단일 목표(신비로운 돌 → 탈출)로 디자인 부담을 최소화하고 플레이어 동기를 집중.

### 4-5. 추상화 구조 설계
자원 / 시설 / 인터랙션 등 공통 동작을 인터페이스·베이스 클래스로 추상화해 새 콘텐츠를 쉽게 추가할 수 있게 구조 설계.

### 4-6. 치팅 툴 시스템 (3주차)
사이클 강제 진행, 신비로운 돌 자동 제작, 자원 조절 등 QA·시연용 인게임 치팅 툴 구현.
