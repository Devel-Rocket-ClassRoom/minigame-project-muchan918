## 1. 게임 개요

**7Days**는 신비로운 제사 부족의 마을 근처에 머물게 된 외지인이 되어, 부족이 요구하는 공물(供物)을 7일마다 바치며 살아남는 **생존 액션 게임**이다.

플레이어는 낮 동안 숲을 탐험하며 나무, 돌, 광석 등의 자원을 채집하고, 동물 사냥 및 제작 시스템을 통해 생존에 필요한 도구를 갖춘다. 밤이 되면 위험 요소가 증가하며, 제한된 시간 안에 오두막으로 복귀해야 한다.

게임은 **7일마다 진행되는 부족의 제사**를 중심으로 진행되며, 플레이어는 제사 전까지 부족이 신께 바칠 공물을 모두 채워 제단에 바쳐야 한다. 공물을 채우지 못하면 **플레이어 자신이 제물로 바쳐지는 연출과 함께 게임 오버**가 된다. 제사가 끝날 때마다 부족은 새로운 신을 모시며, 더 까다로운 공물을 요구해 온다.

## 2. 핵심 게임 루프

```
1일차 시작 (마을 외곽 오두막에서 시작)
    │
    ▼
다가올 7일차 제사 예고 — 요구 공물 목록 전달
    │
    ▼
[낮] 탐험 & 채집 사이클
 ├── 나무 / 돌 / 광석 / 가죽 / 식량 수집
 ├── 도구/장비 제작
 └── 텐트 설치 (임시 거점)
    │
    ▼
[밤] 오두막 복귀 (위험도 증가)
    │
    ▼
7일째 — 제단에 공물 제출 → 제사 의식 진행
    │
  공물 충족?
 ├── YES → 다음 신의 제사 예고 → 다음 사이클 진행
 └── NO  → 제물로 바쳐짐 (게임 오버 연출)
    │
    ▼
최종 제사 → 살아남으면 게임 승리 (부족의 인정)
```

## 3. 구현 범위

### 3-1. 핵심 시스템 (주차별 우선순위 표시)

| 시스템 | 설명 | 우선순위 |
|--------|------|----------|
| 플레이어 이동 & 카메라 | 탑다운 이동, 카메라 추적 | 🔴 1주 (완료) |
| 낮/밤 시간 시스템 | 시간 흐름, 낮↔밤 전환, 날짜 카운트 | 🔴 1주 (대부분 완료) |
| 채집 시스템 | 나무 / 돌 / 풀 / 열매 채집 인터랙션 | 🔴 1주 (완료) |
| 아이템 & 인벤토리 | 아이템 획득, 슬롯 관리 | 🔴 1주 (완료) |
| 체력(HP) 시스템 | HP 관리 + UI, 사망 처리 | 🔴 1주 (완료) |
| **제단(공물) 시스템 — 최소** | **신 1명 + 고정 공물 1세트, 제출/판정만 동작** | 🔴 **1주 (오늘부터 작업)** |
| **7일차 이벤트 트리거** | **Day==7 도달 시 결과 분기, 생존/제물 판정** | 🔴 **1주 (오늘부터 작업)** |
| 배고픔 게이지 | 시간 감소, 섭취 회복, 취침 패널티 | 🟡 2주 |
| 제작(Crafting) 시스템 | 자원 소모 → 도구/장비 제작 | 🟡 2주 |
| 동물 AI & 사냥 | 토끼/사슴/늑대/곰, 상태머신 | 🟡 2주 |
| 조리(모닥불) 시스템 | 날고기 → 구운 고기 | 🟡 2주 |
| 제사 시스템 정교화 | 신 풀, 공물 랜덤 조합, 사이클 진행 | 🟡 2주 |

### 3-2. 탐험 & 생존 요소

| 요소 | 설명 |
|------|------|
| 야생 동물 & 몬스터 AI | 상태 머신 기반 행동 (배회, 추적, 공격, 도망) |
| 위험 지역 탐험 | 날짜 경과에 따라 더 위험한 지역과 자원 등장 |
| 밤 시간 위험도 증가 | 야간 몬스터 활성화, 시야 제한 |
| 오두막 방향 UI | 화면 가장자리에 오두막 방향 표시 |
| 텐트 시스템 | 임시 거점 설치 (야간 안전 확보) |

### 3-3. 제사 시스템

| 요소 | 설명 |
|------|------|
| 공물 요구 | 사이클마다 부족이 다른 신을 모시며 요구 재료가 랜덤 조합으로 결정 |
| 공물 예고 | 사이클 시작 시 부족 장로가 이번 제사의 신과 요구 공물을 알림 |
| 제단 제출 | 마을 입구 제단에 접근해 보유 재료를 제출 |
| 제사 의식 연출 | 7일째 밤 전환 중 부족이 모여 제사를 지내는 연출 |
| 충족 결과 | 부족이 플레이어를 인정 → 다음 사이클 진행 |
| 미달 결과 | 플레이어가 제물로 끌려가 바쳐짐 → 게임 오버 |
| 난이도 증가 | 사이클이 진행될수록 요구 재료 종류/수량 증가 |

### 3-4. 성장 요소

- 도구 업그레이드 (기본 → 강화 → 최상위 티어)
- 오두막 강화 (저장 공간 확장, 작업대 추가)
- 자원 저장 및 관리
- 부족 신뢰도 상승 → 후반 일부 자원 거래 해금 (선택 구현)

## 4. 포트폴리오 핵심 요소

### 4-1. 시간 기반 제사 시스템
낮/밤 변화와 날짜 시스템을 기반으로 7일마다 부족의 제사가 발생하는 구조를 구현하여 게임 진행 흐름과 긴장감을 설계.

### 4-2. 데이터 기반 확장 구조
신/제사 종류, 공물 요구 조합, 몬스터, 자원 등을 ScriptableObject 기반으로 설계하여 새로운 콘텐츠를 쉽게 추가 가능한 구조로 구현.

### 4-3. 세이브 / 로드 시스템
플레이어의 진행 상황을 저장하고 로드하는 시스템 (저장 단위: 게임 내 하루).

### 4-4. 추상화 구조 설계
자원, 몬스터, 제사 이벤트 등의 공통 동작을 추상화(인터페이스 / 베이스 클래스)하여 새로운 콘텐츠를 쉽게 추가할 수 있도록 구조를 설계. 유지보수성과 확장성을 고려한 게임 시스템 구현 경험을 보여주는 것을 목표.

### 4-5. AI 상태 머신
동물과 부족 전사의 행동을 상태 머신(State Machine) 기반으로 구현하여 상황에 따라 자연스럽게 행동하도록 설계.

### 4-6. 치팅 툴 시스템
제사 상황을 원하는 대로 재현할 수 있는 인게임 치팅 툴 구현으로 QA 및 테스트 효율 향상.
